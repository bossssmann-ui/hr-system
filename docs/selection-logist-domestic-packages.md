# Динамические пакеты тестирования для внутренних логистов

Статус: продуктовая спецификация для Phase 14.

Документ расширяет существующую систему отбора логистов. Текущий `logist` остаётся пакетом для международной / мультимодальной логистики. Для логистов, работающих внутри РФ, система должна собирать индивидуальный набор тестов из базового блока и дополнительных модулей по специализациям.

## Цель

Система не должна выдавать всем логистам один и тот же тест. Пакет проверки строится индивидуально на основе:

- данных из резюме;
- ответов кандидата на первичном AI-собеседовании;
- типов грузов, которые кандидат реально возил;
- географии и видов транспорта;
- заявленной зоны ответственности;
- несостыковок между резюме, AI-собеседованием, тестом и живым интервью.

Пример: если кандидат говорит, что основное направление — генгруз по РФ, но иногда он возил негабарит, система назначает основной пакет по автоперевозкам и дополнительный модуль `domestic_oversized_heavy`.

## Общая Логика Отбора

### Шаг 0 — разбор резюме

Парсер резюме извлекает сигналы:

| Сигнал | Примеры |
| --- | --- |
| Вид транспорта | авто, ЖД, контейнеры, море, каботаж, авиа, мультимодал |
| Тип груза | генгруз, сборный, температурный, опасный, негабарит, тяжеловес, оборудование |
| География | ЦФО, Сибирь, Дальний Восток, Север, Якутия, Камчатка, Сахалин, Арктика |
| Операции | подбор перевозчика, маршрутизация, документы, склад, претензии, закрывающие документы |
| Системы | ATI, «Умная Логистика», 1C, TMS/WMS, ЭТРАН, ЭДО |
| Уровень | помощник, диспетчер, логист, ведущий логист, руководитель направления |

Парсер не принимает финальное решение. Он формирует черновой профиль и список тем, которые AI должен уточнить.

### Шаг 1 — первичное AI-собеседование

AI всегда задаёт кандидату базовые вопросы:

1. Какие типы грузов вы реально возили? Назовите именно грузы, а не отрасли клиентов.
2. Какие направления были основными: города, регионы, федеральные округа, труднодоступные территории?
3. Какими видами транспорта вы пользовались в работе?
4. Что было вашей зоной ответственности: только поиск машины, полное ведение рейса, документы, претензии, закрывающие документы?
5. Какие нестандартные перевозки были в вашем опыте?
6. Какие проблемы в перевозках вы решали лично, а не просто передавали подрядчику?

Каждую заявленную специализацию AI классифицирует:

| Уровень | Значение | Как влияет на тест |
| --- | --- | --- |
| `primary` | Основной опыт кандидата. | Полный модуль, высокий вес. |
| `secondary` | Реальный, но нерегулярный опыт. | Короткий модуль, средний вес. |
| `mentioned_only` | Кандидат только сталкивался косвенно. | Уточняющие вопросы без сильного веса. |
| `contradicted` | Резюме и ответы конфликтуют. | Флаг риска и вопросы рекрутеру. |

Важно: этап резюме и этап AI-собеседования с тестированием проходят все откликнувшиеся кандидаты. Система не должна отсеивать кандидата до завершения этих двух этапов, даже если резюме слабое или уже появились первые флаги риска. Ранние флаги накапливаются в профиле и влияют на итоговый балл, но не останавливают прохождение первичной автоматической проверки.

### Шаг 2 — назначение тестов

Каждый внутренний логист получает:

- `domestic_core_operations` — обязательный сквозной блок;
- один или несколько модулей специализации;
- блок проверки на несостыковки и враньё;
- практическое задание, собранное из основной специализации и одной вторичной темы, если она есть.

Правила назначения:

| Триггер в резюме или ответах | Добавить модуль |
| --- | --- |
| FTL, LTL, фуры, сборные, ATI, машины по РФ, генгруз | `domestic_road_ftl_ltl` |
| развозка, маршруты, точки, окна доставки, SLA | `domestic_distribution` |
| ЖД, контейнер, станция, терминал, ЭТРАН | `domestic_rail_container` |
| негабарит, тяжеловес, трал, разрешение, сопровождение | `domestic_oversized_heavy` |
| Север, Якутия, Камчатка, Чукотка, зимник, переправа | `domestic_remote_regions` |
| каботаж, порт РФ, морская линия внутри РФ, Магадан, Сахалин, Камчатка | `domestic_cabotage` |

Если кандидат не заявляет узкую специализацию, система назначает `domestic_core_operations` + `domestic_road_ftl_ltl`.

## 100-Балльная Оценка

Старую модель с фиксированным максимумом по роли нужно заменить на накопительную шкалу 1-100. Балл формируется от первого анализа резюме до завершения AI-собеседования, профтеста и практического задания. Рекрутер видит итоговый балл только после завершения всех автоматических этапов и использует его как один из основных сигналов для решения: звать кандидата на живое интервью или нет.

| Компонент | Баллы |
| --- | ---: |
| Соответствие резюме и AI-собеседования | 15 |
| Сквозные операции внутренней логистики | 20 |
| Основная специализация | 25 |
| Вторичные специализации | 15 |
| Практическое задание | 20 |
| Коммуникация и эскалация | 5 |

Если вторичных специализаций нет, их 15 баллов перераспределяются:

- +10 к основной специализации;
- +5 к практическому заданию.

### Порог допуска к живому интервью

Рекомендуемый hard cutoff: **ниже 70 баллов кандидат не попадает на живое интервью**.

Обоснование:

- живое интервью — самый дорогой этап по времени рекрутера;
- 70 баллов означает, что кандидат подтвердил не только общую логистическую грамотность, но и хотя бы одну рабочую специализацию;
- кандидаты 60-69 часто выглядят частично релевантными, но требуют слишком много ручной проверки;
- редкие специалисты по негабариту, труднодоступным территориям или каботажу могут попадать в ручное исключение, но это не должно быть стандартным автодопуском.

Итоговые зоны:

| Условие | Вердикт |
| --- | --- |
| 85-100, нет RED, максимум 1 ORANGE | Сильный кандидат: рекомендовать живое интервью |
| 70-84, нет RED, максимум 2 ORANGE | Допустить до живого интервью |
| 60-69 | Не допускать автоматически; только ручное исключение HR для редкого опыта |
| Ниже 60 | Отклонить без живого интервью |
| Любой стоп-критерий | Дать завершить первичную автоматическую проверку, затем отклонить |
| 2+ RED | Отклонить |
| 1 RED | Не допускать автоматически; ручная проверка только если RED не является жёсткой ловушкой на ложь |

Если кандидат набрал 70+ баллов, но имеет нерешённый RED-флаг, система не должна автоматически назначать живое интервью. Такой кандидат попадает в ручную проверку HR. Если RED подтверждает враньё, кандидат отклоняется независимо от балла.

Если кандидат набрал ниже 70, но имеет редкий подтверждённый опыт, система может предложить HR ручное исключение. Примеры: реальный опыт каботажа по Дальнему Востоку, негабарита с разрешениями, доставки по зимникам. Такое исключение должно быть явно помечено как `manual_exception_candidate`, чтобы рекрутер понимал, что обычный порог не пройден.

Жёсткие ловушки на ложь:

- кандидат подтверждает опыт с несуществующей TMS, биржей, линией, терминалом или процедурой;
- кандидат заявляет личную ответственность за процесс, но не может объяснить базовую последовательность действий;
- кандидат принимает невозможные условия задачи и не задаёт уточняющих вопросов;
- кандидат говорит «вёл перевозку полностью», но не понимает документы, перевозчика, клиента и закрытие рейса.

## Сквозной Пакет — `domestic_core_operations`

Этот пакет обязателен для всех внутренних логистов. Отдельной специализации по складу не создаём, но понимание связки «склад -> документы -> перевозчик -> клиент -> закрывающие документы» проверяется у каждого кандидата.

### Что Проверяем

- проверка груза до поиска транспорта;
- заявка перевозчику;
- ТН/ТТН/УПД, доверенность, ЭДО, акты;
- повреждения, недостача, пересорт, отказ от приёмки;
- возврат закрывающих документов;
- письменная фиксация изменений;
- коммуникация с клиентом, складом, водителем, перевозчиком и бухгалтерией;
- ранняя эскалация риска.

### Вопросы AI-Собеседования

1. Опишите последнюю перевозку, которую вы вели от заявки до закрывающих документов. Что именно делали вы?
2. Какие данные о грузе вы обязательно уточняете до поиска транспорта?
3. Что должно быть в заявке перевозчику?
4. Какие документы вы проверяете при внутренней перевозке по РФ?
5. Что вы делаете, если склад сообщает о повреждении паллеты перед погрузкой?
6. Что вы делаете, если получатель принял груз с расхождением по количеству мест?
7. Как контролируете возврат закрывающих документов?
8. Когда нужно сообщать клиенту о риске срыва доставки?

### Вопросы ПрофТеста

1. Что нельзя запускать в перевозку без уточнения?
   - Только стоимость перевозки.
   - Вес, габариты, количество мест, адреса, сроки, требования к погрузке и выгрузке.
   - Только ФИО водителя.
   - Только марку автомобиля.
   Correct: Вес, габариты, количество мест, адреса, сроки, требования к погрузке и выгрузке.

2. Если груз повреждён на складе до погрузки, логист должен:
   - Всё равно отправить, чтобы не сорвать срок.
   - Зафиксировать повреждение, уведомить ответственных, согласовать дальнейшие действия.
   - Попросить водителя решить на месте.
   - Удалить повреждённое место из документов.
   Correct: Зафиксировать повреждение, уведомить ответственных, согласовать дальнейшие действия.

3. Что является плохим признаком в работе логиста?
   - Письменное подтверждение изменения ставки.
   - Фотофиксация спорной ситуации.
   - Устная договорённость о доплате без фиксации.
   - Предупреждение клиента о риске заранее.
   Correct: Устная договорённость о доплате без фиксации.

4. Открытый вопрос: машина прибыла на выгрузку, получатель не принимает груз из-за ошибки в документах. Опишите действия.

5. Открытый вопрос: клиент просит «просто отправить быстрее», но не дал точный вес и габариты. Как отвечаете?

### Вопросы Рекрутеру

1. Назовите последний случай, когда закрывающие документы вернулись с проблемой. Как решали?
2. Что вы лично проверяете в заявке перевозчику?
3. Расскажите ситуацию, когда пришлось спорить с перевозчиком по простою или доплате.
4. Как вы отличаете проблему склада от проблемы перевозчика?

## Пакет — `domestic_road_ftl_ltl`

### Что Проверяем

- FTL/LTL и сборные грузы;
- подбор типа машины;
- проверка перевозчика;
- работа с ATI или аналогичными площадками;
- контроль рейса;
- срыв подачи, замена машины, простой, переадресация;
- температурные, хрупкие и габаритные ограничения, если они есть.

### Вопросы AI-Собеседования

1. Какие типы машин вы подбирали чаще всего и под какие грузы?
2. Как проверяете перевозчика перед постановкой на рейс?
3. Чем FTL отличается от LTL в вашей практической работе?
4. Что делаете, если машина сорвала подачу за 2 часа до погрузки?
5. Какие данные нужны, чтобы понять, подойдёт ли тент, реф, борт или манипулятор?
6. Какие проблемы чаще всего возникают на внутренних автоперевозках?

### Вопросы ПрофТеста

1. LTL означает:
   - Полная машина под одного клиента.
   - Сборная перевозка, где груз занимает часть транспорта.
   - Только перевозка негабарита.
   - Только городская доставка.
   Correct: Сборная перевозка, где груз занимает часть транспорта.

2. Перевозчик просит доплату за простой. Первое действие логиста:
   - Сразу согласовать оплату.
   - Проверить заявку, время прибытия, отметки, переписку и причину простоя.
   - Отказать без проверки.
   - Переложить решение на водителя.
   Correct: Проверить заявку, время прибытия, отметки, переписку и причину простоя.

3. Для выбора машины важнее всего:
   - Только расстояние.
   - Вес, объём, габариты, тип загрузки, условия перевозки.
   - Только ставка.
   - Только город отправления.
   Correct: Вес, объём, габариты, тип загрузки, условия перевозки.

4. Открытый вопрос: перевозчик сообщает, что машина сломалась на маршруте. Клиент ждёт груз завтра утром. Ваши действия?

5. Открытый вопрос: клиент хочет отправить сборный груз, но требует доставку как FTL. Как объясните варианты?

### Практическое Задание

Клиенту нужно доставить 8 паллет, 3 400 кг, Москва -> Екатеринбург. Срок: послезавтра утром. Клиент не знает точные габариты паллет и говорит, что «обычная фура точно подойдёт». Перевозчик найден на бирже, просит предоплату и не готов показать рекомендации.

Кандидат должен:

- запросить недостающие данные;
- сравнить FTL/LTL;
- проверить перевозчика;
- зафиксировать условия в заявке;
- предупредить о риске срока;
- предложить резервного перевозчика.

Ловушки:

- соглашается без точных габаритов;
- подтверждает сомнительного перевозчика без проверки;
- обещает срок без проверки окон погрузки и транзита.

### Вопросы Рекрутеру

1. Как вы проверяли перевозчиков на ATI или аналогичных площадках?
2. Расскажите пример, когда приходилось срочно менять машину.
3. Что пишете в заявке, чтобы защититься от спора по простою?
4. Когда сборная перевозка не подходит, даже если она дешевле?

## Пакет — `domestic_distribution`

### Что Проверяем

- маршруты с несколькими точками;
- окна доставки;
- городская и региональная развозка;
- SLA и приоритеты;
- координация водителей;
- возвраты, отказы, частичная приёмка;
- документы от нескольких получателей.

### Вопросы AI-Собеседования

1. Сколько точек в день вы обычно вели на одного водителя или маршрут?
2. Как строили маршрут: вручную, в TMS, через таблицы, через карты?
3. Что делаете, если одна точка задерживает весь маршрут?
4. Как работаете с окнами доставки?
5. Как фиксируете отказ получателя принять груз?
6. Как контролируете документы после развозки по нескольким точкам?

### Вопросы ПрофТеста

1. При развозке с окнами доставки первым нужно учитывать:
   - Только километраж.
   - Временные окна, приоритеты, ограничения точек и реальный порядок движения.
   - Только стоимость топлива.
   - Только пожелания водителя.
   Correct: Временные окна, приоритеты, ограничения точек и реальный порядок движения.

2. Если одна точка не принимает груз и задерживает маршрут, логист должен:
   - Ждать до конца дня без уведомлений.
   - Оценить влияние на следующие точки, уведомить заинтересованных, согласовать решение.
   - Отменить все следующие доставки.
   - Попросить водителя выбрать самому.
   Correct: Оценить влияние на следующие точки, уведомить заинтересованных, согласовать решение.

3. Главный риск при многоадресной доставке:
   - Только пробки.
   - Срыв окон, потеря документов, частичные отказы, накопление задержек.
   - Только цена рейса.
   - Только марка автомобиля.
   Correct: Срыв окон, потеря документов, частичные отказы, накопление задержек.

4. Открытый вопрос: 15 точек, 2 машины, три клиента принимают только до 12:00, одна машина задержалась на складе. Что делаете?

5. Открытый вопрос: получатель на одной точке принял не весь груз. Какие документы и действия нужны?

### Практическое Задание

Есть 18 точек по Владивостоку и пригородам, 3 машины, два клиента принимают только до 11:30, один клиент требует оригиналы документов, одна машина сломалась после первой точки.

Кандидат должен:

- перестроить маршрут;
- перенести или разделить точки;
- уведомить клиентов;
- зафиксировать отказ или задержку;
- защитить возврат документов.

Ловушки:

- оптимизирует только по расстоянию;
- игнорирует окна доставки;
- не отслеживает оригиналы документов.

### Вопросы Рекрутеру

1. Как решали конфликт между коротким маршрутом и окнами доставки?
2. Что делаете, если водитель не успевает на последние точки?
3. Как фиксируете частичный отказ или недовоз?
4. Как контролировали возврат документов при большом количестве точек?

## Пакет — `domestic_rail_container`

### Что Проверяем

- ЖД и контейнерные перевозки по РФ;
- станции, терминалы, операторы;
- наличие контейнера;

## Динамическая Логика AI-Собеседования

### Ветвление По Типу Груза

Если кандидат говорит «генгруз», AI уточняет:

- какой именно груз: паллеты, коробки, оборудование, стройматериалы, ТНП, сырьё;

## Проверка На Враньё

### RED-Флаги

- Кандидат подтверждает опыт с несуществующей системой, биржей, портовой линией или терминалом.
- Кандидат заявляет специализацию, но не может объяснить базовую последовательность действий.
- Кандидат принимает невозможные условия задачи без уточнений.
- Кандидат говорит «вёл рейс полностью», но не понимает документы, перевозчика, клиента или закрытие.
- Кандидат заявляет опыт негабарита, но не может назвать ни одного реального веса/габарита.

### ORANGE-Флаги

- Ответы общие, без маршрутов, груза, цифр, документов и решений.
- Кандидат использует термины, но не объясняет последствия.
- Кандидат заявляет полный цикл, но описывает только узкое действие.
- Резюме сильное, а тест по той же теме слабый.
- L-шкала психотеста показывает риск социально желательных ответов.

## Нагрузка, Слоты И Одновременные Собеседования

Обычные этапы тестирования не требуют назначения даты и времени. Анкета, профтест и практическое задание должны быть асинхронными: кандидат проходит их по ссылке в допустимый период, а сервер сохраняет ответы по сессии.

Риск лагов возникает не от самого факта двух кандидатов одновременно, а от AI-части:

- одновременные запросы к LLM-провайдеру;
- внешние rate limits и временные ошибки модели;
- длинные AI-собеседования с несколькими последовательными вопросами;
- генерация индивидуальных тестов и итогового отчёта;
- текущая development-очередь в проекте является in-memory stub и не должна считаться production-механизмом управления нагрузкой.

### Правило По Умолчанию

Система должна разрешать всем откликнувшимся проходить асинхронные этапы без записи на конкретное время. Назначение даты и времени требуется только для этапов, где AI работает в режиме синхронного диалога или где нагрузка может ухудшить опыт кандидата.

Рекомендуемая модель:

| Этап | Нужна запись на слот | Почему |
| --- | --- | --- |
| Разбор резюме | Нет | Выполняется фоном, можно поставить в очередь. |
| Анкета | Нет | Простая форма, почти не нагружает систему. |
| ПрофТест | Нет | Таймер серверный, ответы сохраняются в БД. |
| Практическое задание | Нет | Кандидат пишет ответ асинхронно. |
| AI-собеседование в формате чата | Да, если ожидается высокая нагрузка или LLM работает медленно | Много последовательных LLM-вызовов, важен стабильный UX. |
| Живое интервью с рекрутером | Да | Это человеческий календарь. |

### Capacity Guard

Перед запуском AI-собеседования система должна проверять доступную AI-ёмкость:

- максимум активных AI-собеседований одновременно;
- максимум LLM-запросов в минуту;
- максимум LLM-запросов на одного кандидата;
- таймаут ответа модели;
- повтор запроса при временной ошибке;
- graceful fallback: «AI сейчас занят, выберите ближайший слот».

Начальная настройка для MVP:

- `max_active_ai_interviews = 3`;
- `max_llm_requests_per_minute = provider_limit * 0.7`;
- `ai_response_timeout_sec = 30`;
- `ai_interview_slot_duration_min = 30`;
- `ai_interview_buffer_min = 5`.

Значения должны быть конфигурируемыми, потому что реальные лимиты зависят от выбранного LLM-провайдера и тарифа.

### Когда Включать Запись На AI-Собеседование

Запись на дату и время нужна, если выполняется хотя бы одно условие:

- AI-собеседование проводится как последовательный чат в реальном времени;
- одновременно запускается больше настроенного лимита активных AI-интервью;
- LLM-провайдер возвращает rate limit или частые timeout;
- среднее ожидание ответа AI выше 5-7 секунд;
- компания проводит массовый набор и отправляет десятки приглашений в один день;
- рекрутеру важно распределить поток кандидатов по дням.

Если AI-собеседование реализовано не как live-chat, а как форма с динамически подобранными вопросами и последующей фоновой оценкой, запись на слот не обязательна. В этом случае достаточно очереди, лимитов и статуса «оценка обрабатывается».

### Что Видит Кандидат

Если ёмкость свободна:

- кандидат начинает AI-собеседование сразу;
- видит прогресс и ограничение по времени;
- при временной ошибке AI получает понятное сообщение без потери уже данных ответов.

Если ёмкость занята:

- кандидат выбирает ближайший свободный слот;
- система показывает часовой пояс;
- кандидат получает ссылку на сессию;
- слот освобождается при неявке через заданный grace period.

### Что Видит Рекрутер

Рекрутер должен видеть:

- сколько кандидатов сейчас проходит AI-собеседование;
- очередь ожидания;
- кандидатов со статусом `ai_processing`;
- ошибки AI-оценки;
- кандидатов, которым нужно переназначить слот.

## Рекомендации Для Реализации

Рекомендуемая модель:

- верхнеуровневая роль остаётся `logist`;
- в сессию отбора добавляется профиль `assessment_profile` или `specializations`;
- выбранные пакеты хранятся с уровнем `primary`, `secondary`, `mentioned_only`, `contradicted`;
- баллы хранятся по модулям и затем агрегируются в 100-балльную оценку;
- первые два этапа — разбор резюме и AI-собеседование с тестированием — создаются для всех откликнувшихся без предварительного отсева;
- автоматическая отсечка на живое интервью применяется только после завершения первичной автоматической проверки;
- порог живого интервью хранится конфигурируемо, но значение по умолчанию — 70 баллов;
- AI-собеседования должны иметь capacity guard: лимит активных сессий, лимит LLM-запросов, таймауты и возможность записи на слот;
- для production заменить in-memory queue на durable queue с управлением concurrency и retry;
- итоговый вердикт должен включать индивидуальные вопросы для рекрутера.

Не стоит превращать каждую внутреннюю специализацию в отдельную роль, если это не требуется для отчётности по вакансиям. Один кандидат может одновременно иметь опыт авто, ЖД, негабарита, труднодоступных регионов и каботажа, поэтому правильнее использовать динамические пакеты внутри роли `logist`.
